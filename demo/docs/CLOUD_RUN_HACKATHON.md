## üèóÔ∏è Architecture Deep Dive

### Component Breakdown

#### 1. Cloud Run Frontend (Service)
**Container**: `agenticdid-frontend:latest`
**Port**: 8080 (Cloud Run standard)
**Tech Stack**:
- React 18 with TypeScript
- Vite bundler (ultra-fast)
- TailwindCSS for styling
- Lucide icons

**Features**:
- Results-focused UX (user picks goal, not agent)
- Real-time verification timeline
- ZKP proof visualization
- Proof log modal with audit trail

**Scaling**: Auto-scales 0-100 instances based on traffic

#### 2. Cloud Run Backend (Service)
**Container**: `agenticdid-api:latest`
**Port**: 8080 (mapped from 8787)
**Tech Stack**:
- Fastify web framework
- Bun runtime (3x faster than Node)
- TypeScript strict mode

**Endpoints**:
- `POST /api/challenge` - Generate verification challenge
- `POST /api/verify` - Verify agent credentials
- `GET /health` - Health check for Cloud Run

**Scaling**: Independent scaling from frontend

#### 3. Google ADK Agents (MCP Protocol)

**Agent Architecture**:
```typescript
// Each agent is a Google ADK agent with MCP communication
class CometAgent extends ADKAgent {
  // Orchestrator agent - coordinates others
  async selectAgent(userIntent: string): Promise<Agent> {
    // Uses Gemini to understand intent
    // Returns appropriate specialist agent
  }
  
  async verifyMutualAuth(): Promise<boolean> {
    // Midnight ZKP verification
  }
}

class BankerAgent extends ADKAgent {
  role: 'finance'
  scopes: ['read:balance', 'transfer:funds']
  
  async executeTask(task: Task): Promise<Result> {
    // MCP communication with bank services
    // Midnight credential verification
  }
}
```

**MCP Communication Flow**:
```
User Intent
  ‚Üì (MCP message)
Comet Agent (orchestrator)
  ‚Üì (analyzes intent with Gemini)
  ‚Üì (selects specialist)
Banker/Shopper/Traveler Agent
  ‚Üì (MCP to service)
External Service (Bank/Amazon/Airline)
  ‚Üì (Midnight ZKP verification)
Success/Failure
```

#### 4. Midnight Network (Privacy Layer)

**Compact Smart Contracts** (deployed to Midnight devnet):

1. **AgenticDIDRegistry.compact**
   - Stores agent DID ‚Üí public key mappings
   - Private credential storage
   - Revocation list (bitmap)

2. **CredentialVerifier.compact**
   - Verifies ZK proofs
   - Checks expiry timestamps
   - Validates signatures

3. **DelegationChain.compact**
   - Tracks User ‚Üí Agent delegations
   - Scope enforcement
   - Time-based expiry

**Why Midnight + Google Cloud**:
- Cloud Run: Scalable, serverless deployment
- Midnight: Privacy-preserving verification
- Best of both: Performance + Privacy

## üöÄ Deployment Plan

### Phase 1: Cloud Run Setup (Day 1)

1. **Activate Google Cloud Account**
   ```bash
   gcloud auth login
   gcloud config set project agenticdid-io
   ```

2. **Enable Cloud Run API**
   ```bash
   gcloud services enable run.googleapis.com
   gcloud services enable cloudbuild.googleapis.com
   ```

3. **Deploy Frontend**
   ```bash
   cd apps/web
   gcloud run deploy agenticdid-frontend \
     --source . \
     --platform managed \
     --region us-central1 \
     --allow-unauthenticated
   ```

4. **Deploy Backend**
   ```bash
   cd apps/verifier-api
   gcloud run deploy agenticdid-api \
     --source . \
     --platform managed \
     --region us-central1 \
     --allow-unauthenticated
   ```

### Phase 2: Google ADK Integration (Day 2)

1. **Install Google ADK & Gemini CLI**
   ```bash
   npm install @google-cloud/agent-development-kit
   # Install Gemini CLI for development assistance
   gcloud components install gemini
   ```

2. **Use AI Studio for Agent Design**
   - Open https://aistudio.google.com
   - Design agent prompts and behaviors
   - Test agent responses with different scenarios
   - Export prompts for integration
   - Share app link for hackathon submission

3. **Convert Agents to ADK Format (with Gemini CLI assistance)**
   ```bash
   # Use Gemini CLI to accelerate development
   gemini code generate --prompt "Convert TypeScript agent to Google ADK format"
   ```
   - Comet ‚Üí ADK Orchestrator Agent
   - Banker ‚Üí ADK Financial Agent
   - Shopper ‚Üí ADK Commerce Agent
   - Traveler ‚Üí ADK Booking Agent

4. **Implement MCP Communication**
   - Agent-to-agent messaging
   - Service integration points
   - Error handling and retries

### Phase 3: Midnight Integration (Day 3)

1. **Deploy Compact Contracts**
   ```bash
   # Deploy to Midnight devnet
   midnight-cli deploy contracts/AgenticDIDRegistry.compact
   midnight-cli deploy contracts/CredentialVerifier.compact
   midnight-cli deploy contracts/DelegationChain.compact
   ```

2. **Connect Cloud Run ‚Üí Midnight**
   - Environment variables for contract addresses
   - Midnight provider setup
   - ZKP verification hooks

### Phase 4: Testing & Demo (Day 4)

1. **End-to-end testing**
2. **Record 3-minute demo video**
3. **Write blog post**
4. **Social media promotion**
